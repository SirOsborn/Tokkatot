# Specifications & Requirements

This document captures the current requirements for the Tokkatot cloud upgrade and hardware rollout.

---

## 1) Product Scope
- Smart poultry management focused on **coop-level automation**.
- **Farm** is a container; each farm can have multiple coops.
- Each coop can have different hardware, thresholds, and schedules.

---

## 2) Roles & Access Control
- **Owner (Farmer)**: full control of farms, coops, devices, schedules, and workers.
- **Worker**: can view + change configuration (same as owner except account admin).
- **Viewer**: read-only (monitoring/alerts only).

---

## 3) Device Catalog (Per Coop)
Each coop can have any subset of these 6 device types. Missing devices are allowed and shown as **inactive**.

**Actuators (Scheduled/Manual)**
- `feeder_motor` (high-torque motor, on/off)
- `conveyor_belt` (on/off)
- `fan` (on/off)
- `heater` (on/off)

**Sensors (Monitor-only)**
- `temp_humidity`
- `water_level`

---

## 4) Automation & Control Rules
1. **Schedules (Actuators)**
   - Daily start time
   - Run for X minutes
   - Off for Y minutes
   - Repeat sequence (coop-level)
2. **Temperature Thresholds (Coop)**
   - If temp < min → heater ON
   - If temp > max → fan ON
   - Else both OFF
3. **Water Alert Rule (Coop)**
   - If water level below **half threshold** for **1 minute**, alert farmer.

---

## 5) Cloud & Connectivity
- **Raspberry Pi 4B** is the gateway between ESP32 and the cloud.
- Pi posts telemetry and device list to cloud.
- Pi executes schedules locally (works offline if internet drops).

Required cloud endpoints:
- `POST /v1/farms/:farm_id/coops/:coop_id/devices/report`
- `POST /v1/farms/:farm_id/coops/:coop_id/telemetry`
- `GET /v1/farms/:farm_id/coops/:coop_id/temperature-timeline`

---

## 6) Registration & Verification
- **Registration Key** is the only verification path.
- No SMS/email verification endpoints are required.
- Keys are generated by the Tokkatot team at purchase/install time.

---

## 7) Data & Monitoring Requirements
- **Current** temperature/humidity shown in the dashboard.
- **Today** view: hourly low/high.
- **History** view: daily low/high (yesterday, previous days).
- Coop selectors control which data is displayed.

---

## 8) Non-Functional Requirements
- Mobile-first UI, Khmer-first labels.
- High-contrast UI for visibility in bright farm environments.
- API must remain responsive under intermittent connectivity.
- System must tolerate missing devices without errors.
